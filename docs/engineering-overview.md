# Engineering Docs

This `docs/` directory is now human-oriented.

## Principle

- Human docs explain product behavior, operating model, and high-level
  engineering decisions.
- Agent skills hold repeatable code-authoring and review instructions.

## Agent skills

Use these for implementation guidance:

- `skills/checa-repo-workflow/SKILL.md`
- `skills/checa-fsd-architecture/SKILL.md`
- `skills/checa-api-validation/SKILL.md`
- `skills/checa-query-cache/SKILL.md`
- `skills/checa-notification-workflows/SKILL.md`
- `skills/checa-react-ui-conventions/SKILL.md`

## Keep in docs

Good candidates for `docs/`:

- product and business behavior
- architecture summaries for humans
- onboarding and environment setup
- operational runbooks
- migration notes that humans need to read

## Keep in skills

Good candidates for `skills/`:

- code placement rules
- review checklists
- route and validation patterns
- query and cache conventions
- UI implementation conventions
