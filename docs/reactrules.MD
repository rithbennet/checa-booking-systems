# React UI Overview

This document is for humans. Detailed component authoring rules now live in repo
skills for agents.

## What matters

- The frontend favors small focused components.
- Component APIs should stay clear and contextual.
- Derived values should usually stay derived rather than becoming local state.

## Where to look

- For repo workflow: `skills/checa-repo-workflow/SKILL.md`
- For React and UI agent guidance: `skills/checa-react-ui-conventions/SKILL.md`

## When to update this document

Update this file only when the human-level UI engineering philosophy changes,
not when individual implementation examples or local conventions change.
